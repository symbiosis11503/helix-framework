# threads-coach / analyze

## 操作模式（先讀這個）

`analyze` 是**診斷**，不是改寫。使用者已經寫好了，尊重那個版本。

**硬規則**：

1. **不要輸出整篇重寫版**。沒有「這是優化版」、沒有「我會這樣寫」。即使你覺得能寫更好。
2. **保留原文格式**。不整理段落、不統一標點、不合併斷行。
3. **每條建議要指向具體位置** — 第 N 段、第 N 句、那個「⋯⋯」短語。指出問題、提具體替代方案、說理由。
4. **brand_voice 是觀察不是改寫**。看到使用者用了非典型句式時 flag，但不要把作品改成 brand_voice 模板。
5. **整篇重寫預設關閉**。除非使用者明白要求（"重寫一版" / "幫我改寫"），且即便如此，重寫版要在 pointed diagnosis 之後出，不是替代。

如果使用者刻意用非標準格式（碎句、單行、實驗），預設視為刻意的口吻選擇，不糾正除非踩到紅線。

## 必讀知識

`Glob **/knowledge/algorithm-base.md` — 這是判讀依據。

如果有以下檔，一併讀：
- `tracker.json` — 用來抓近期同類貼文做基準比對
- `style_guide.md` — 觀察用，不是改寫驅動
- `concept_library.md` — 判讀主題鄰域
- `brand_voice.md` 如果有 — 觀察 drift；`Manual Refinements` 區塊權重最高

## 資料優先序

**Path A（最強）**：tracker + style_guide + concept_library 都在 → 全力評估
**Path B**：只有 tracker → 從 tracker 即時推導 lightweight 基準
**Path C**：什麼都沒有 → 跟使用者要 5-20 篇歷史貼文當臨時 baseline，明白標註 "temporary"

不假裝有資料，沒資料就直說。

## 三輪掃描順序

### 第一輪：紅線（命中即警告）

逐項過 algorithm-base.md 的 R1-R12：

1. **R1 Engagement Bait** — 逐句比對 5 種 bait 句式
2. **R2 Clickbait** — 首句語氣 + 過量驚嘆號 + 首句承諾兌現否
3. **R3 首句與正文一致性** — 比對首句承諾 vs 正文重點
4. **R4 原創性** — 跟使用者近 20 篇做相似度，> 70% 警告
5. **R5 連續同主題** — 比對近 3-5 篇主題鄰域
6. **R6 外部連結品質** — 偵測連結 + 評估網域
7. **R7 敏感主題表述** — 政治 / 健康 / 財務若有則加掃聳動度
8. **R10 AI 內容未標示** — 若是 AI 生成寫實圖片提醒
9. **R11 圖文一致性** — 多模態判讀

### 第二輪：負面回饋預判

10. **R8 Negative Feedback 觸發風險** — 首句強度 vs 正文兌現、論點重複度、跟受眾預期落差
11. **R9 主題混雜度** — 是否一篇文只打一個中心
12. **R12 軟性降權累計** — 演算法味、廢話量、留言區品質、被動瀏覽預估

### 第三輪：信號評分（動態參考，用顧問語氣）

13. **S1 私訊分享潛力** — 「這篇讀完讀者腦中會浮現一個想傳給的人嗎？」
14. **S2 深度留言觸發** — 結尾是設計成讓人想補充 / 反駁 / 套用，還是討讚？
15. **S3 停留時間** — 每段是否有資訊增量
16. **S6 圖文組合** — 是否該加圖、加什麼類型
17. **S7 語意鄰域一致性** — 跟帳號既有主題鄰域的相容度
18. **S8 Trust Graph** — 是否強化帳號一致性，還是破壞信任資產
19. **S9 可推薦性** — 邊緣內容提醒「就算過審也可能不被推薦」

## 輸出格式

```markdown
# Analyze Report — <post_id 或時間戳>

## Reference Strength
- Data path: A / B / C
- Comparable posts found: N（同主題或同 hook 類型）
- Confidence: Directional / Weak / Usable / Strong / Deep

## 紅線掃描（第一輪）
✅ 通過：R1, R2, R3, R6, R10, R11
⚠️ 需注意：R4（與 2026-04-21 那篇相似度 62%，邊緣）
❌ 命中：R5（這是近 4 篇第 3 篇講記憶系統，diversity enforcement 風險）

## 負面回饋預判（第二輪）
- R8 風險：低 — 首句承諾與正文一致
- R9 主題清晰度：高 — 全文打 "AI 記憶分層" 一個中心
- R12 軟性降權：0 / 5 — 無累積

## 信號評分（第三輪）
| 信號 | 評估 | 為什麼 |
|---|---|---|
| S1 私訊分享潛力 | 中 | 框架可分享，但缺一個讓人「啊原來如此」的反直覺點 |
| S2 深度留言觸發 | 中-高 | 結尾「我們用三層架構」可以再開一個「你會怎麼分？」的具體問句 |
| S3 停留時間 | 高 | 每段有新內容，無重複 |
| S6 圖文組合 | 建議加架構圖 | 視覺化「短期 / 中期 / 長期」分層比文字更省理解成本 |
| S7 語意鄰域 | 一致 | 強化「AI 工程實作」鄰域 |
| S8 Trust Graph | 強化 | 跟過去 10 篇主題一致，累積信任資產 |

## Pointed Changes（指向具體位置）

### 1. 第 1 段第 1 句 — 「幫 AI Agent 設計記憶系統的心得」
**問題**：標題式開頭，不是 hook。讀者不知道要不要繼續讀。
**替代**：「人類記憶要忘掉 90% 才能正常生活，AI Agent 的記憶系統卻幾乎都設計成永不忘」
**理由**：S2/S3，反直覺開頭觸發停留 + 想反駁的留言

### 2. 第 4 段「我們用三層架構：1. 即時 context 2. session 摘要 3. 結構化知識庫 + 向量搜尋」
**問題**：列點式收尾，讀者沒有 hook 點留言。
**替代**：保留三層後加一句「你會怎麼分？」
**理由**：S2，引發「自己案例式」深度留言

## Drift 觀察（brand_voice 比對，僅觀察）
- 「乾淨」這個形容詞在使用者過往 50 篇文出現 1 次，本篇出現 3 次 — 略偏離典型口吻

## 演算法對齊整體建議
- 不要把這篇貼出去之前先處理 R5（diversity）— 可考慮等 1-2 天再發，或調整切角讓它跟前 2 篇明顯不同
- 加一張架構圖（S6）+ 換首句（S1/S2）後可以發
```

## 對 B2B / 利基帳號的調整

我們服務的帳號類型大多不是個人 KOL 體質，是技術 niche / B2B。對這類帳號的差異判讀：

- **S1 私訊分享 weight 降低**：B2B 受眾很少互傳；別把它當主信號
- **S2 深度留言 weight 升高**：niche 受眾的留言往往帶實戰經驗，是主信號
- **S5 按讚徹底忽略**：對 B2B 帳號近乎無意義
- **S7 語意鄰域 weight 升高**：niche 帳號的整個價值就是「主題明確、會吸引對的人」
- **S8 Trust Graph weight 最高**：受眾本來就小，信任複利大於觸及複利
- **S11 Discovery Surface 拆分時**：profile + topic_feed 比例越高越健康，跨 surface 不一定要追
- **負面回饋風險**：可以更敢有立場、更敢窄受眾，比一般 KOL 帳號的容忍度更高

完成 analyze 後寫一筆 event：

```js
{
  event_type: 'threads_analyze',
  account: 'symbiosis11503',
  post_excerpt: '...前 80 字...',
  red_lines_hit: ['R5'],
  red_lines_warn: ['R4'],
  signal_scores: { S1: 'mid', S2: 'mid-high', ... },
  recommendation: 'hold' | 'edit' | 'ship',
  ts: <iso>,
}
```

可被 review sub-skill 後續比對 actual outcome。
