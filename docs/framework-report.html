一句話判斷標準

一個成熟的 Agent Framework 不是看它「會不會叫 LLM」，而是看它有沒有同時處理：

  • 會做事 — 工具 + 技能 + 任務 + 工作流
  • 記得住 — 對話記憶 + 長期記憶 + 知識治理
  • 管得住 — 安全 + 權限 + 行為治理
  • 查得到 — 追蹤 + 審計 + 監控

1 推理引擎 Reasoning Engine

處理 Agent 怎麼思考、怎麼決策

基礎能力

  • 單輪問答 — 接收問題，回傳答案
  • 多輪對話 — Session-aware，保持對話上下文
  • 結構化輸出 — 依 JSON schema 回傳格式化結果

進階推理

  • Plan-Act-Observe 循環 — 計劃→執行→觀察→決策，反覆直到完成
  • 任務拆解 — 將複雜任務分解為子目標
  • 反思與自我修正 — 根據執行結果調整策略
  • 多步推理 — 長鏈推理能力

模型管理

  • 多模型路由 — 依成本/能力選擇不同模型
  • Provider 切換 — 支援多個 LLM 供應商
  • Fallback 策略 — 主模型失敗時自動切換

2 工具系統 Tool System

處理 Agent 能用哪些工具、怎麼呼叫

工具生命週期

  • 動態註冊/反註冊 — Runtime 新增或移除工具
  • 參數 Schema 定義 — 每個工具有明確的輸入格式
  • 工具描述 — LLM 可讀的工具說明
  • 工具分級 — L1 讀取 / L2 寫入 / L3 破壞性

執行機制

  • 執行 Pipeline — 前置檢查 → 權限驗證 → 執行 → 後處理
  • Hook 攔截 — Before hook 可中止，After hook 可修改結果
  • 能力綁定 — 工具與角色/能力的對應關係
  • 結果格式化 — 工具輸出標準化

擴展性

  • MCP 協議 — 外部工具伺服器動態發現
  • Plugin 機制 — 第三方工具包載入
  • Manifest 匯出 — OpenAI/標準格式的工具清單

處理 Agent 會什麼、怎麼學新技能

技能定義

  • 技能格式 — Markdown 模板 / Prompt template / 程式碼
  • Frontmatter 元資料 — 名稱、描述、參數、標籤、前置條件
  • 分類與標籤 — Category / Tag 組織

技能生命週期

  • 自動發現 — 掃描目錄自動載入

  • 版本管理 — 技能更新與版本追蹤
  • 技能創造 — Agent 自動產生新技能
  • 技能市集 — 共享與發布技能包

4 對話記憶 Session Memory

處理當前對話的上下文管理

Session 管理

  • Session 建立/切換/關閉
  • Context Window 管理 — Token budget 分配
  • 多 Agent Session 隔離

壓縮策略

  • LLM 摘要壓縮 — 用 LLM 生成結構化摘要
  • Head/Tail 保護 — 保留開頭和結尾訊息
  • Session Chain — 壓縮後新 session 銜接舊 session
  • Tool Output 裁剪 — 壓縮前先截斷冗長工具輸出

5 長期記憶 Long-term Memory

處理跨 Session 的持久化記憶

記憶分層

  • Episodic（情節記憶） — 發生過的事件和經驗
  • Semantic（語義記憶） — 結構化知識和事實
  • Procedural（程序記憶） — 學到的工作流和步驟

檢索機制

  • 全文搜尋 — FTS5 / tsvector / ILIKE
  • 向量語義搜尋 — Embedding + ANN index
  • 混合搜尋 — 全文 + 向量結合
  • Tag 檢索 — 標籤匹配

記憶動態

  • 重要性評分 — 0.0~1.0 評分
  • 時間衰減 — 記憶隨時間降權
  • 記憶合併 — 相關記憶合併去重
  • 自動提取 — 從對話自動擷取重要事實
  • Context 注入 — 記憶自動注入 LLM prompt

  • 驗證 — 規則檢查（斷鏈/孤頁）+ 語義檢查（矛盾/過期）

品質保證

  • 來源追溯 — Provenance tracking
  • 可信度分級 — Reliability tier
  • 去重 — 重複偵測與合併
  • Two-layer Lint — 先確定性檢查，再 LLM 語義檢查

7 任務管理 Task Management

處理任務的完整生命週期

基礎

  • 任務生命週期 — pending → running → done / failed
  • 優先序 — critical / high / normal
  • 佇列管理 — FIFO + 優先序排序
  • 進度回報 — 即時狀態更新

長任務

  • 超時偵測 — 自動暫停超時任務
  • 重試策略 — 可配置重試次數與間隔
  • 部分結果 — 失敗前已完成的步驟保留

輸出防護

系統防護

  • CSP Headers — Content Security Policy
  • WebSocket 認證 — 未授權連線拒絕
  • Secret 管理 — 環境變數，不硬編碼
  • Sandbox 隔離 — 執行環境隔離

16 行為治理 Behavior Governance

處理 Agent 的自主邊界

自主模式

  • 關閉 (Off) — Agent 不主動行動
  • 建議 (Suggest-only) — 只建議，不執行
  • 有限自主 (Bounded) — 在規則內自主執行

控制機制

  • Discovery Rules — 自主發現規則 + 冷卻時間
  • 自優化提案 — 記錄但不直接改寫
  • Hook 攔截 — 生命週期攔截點
  • Kill Switch — 緊急停止開關
  • 行為預算 — 自主行為的範圍限制
  • Human Override — 人類隨時可接管
